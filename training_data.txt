 <eos>
Credential Guard and Virtualization-based Security (VBS) Key Isolation are security features in Windows designed to protect sensitive credentials from theft. Here's a breakdown: <eos>
 <eos>
 <eos>
 <eos>
Credential Guard: <eos>
 <eos>
 <eos>
 <eos>
Purpose: <eos>
 <eos>
It's a security feature that helps prevent credential theft attacks, such as "pass-the-hash" and "pass-the-ticket." <eos>
 <eos>
It does this by isolating sensitive credentials, like NTLM password hashes and Kerberos Ticket Granting Tickets (TGTs), so that only privileged system software can access them. <eos>
 <eos>
How it works: <eos>
 <eos>
Credential Guard uses Virtualization-based Security (VBS) to create an isolated environment where these sensitive credentials are stored and protected. <eos>
 <eos>
Virtualization-based Security (VBS) Key Isolation: <eos>
 <eos>
 <eos>
 <eos>
Purpose: <eos>
 <eos>
VBS is a hardware-assisted virtualization technology that creates a secure, isolated region of memory. <eos>
 <eos>
Key Isolation is a part of VBS that specifically isolates cryptographic keys and other sensitive data. <eos>
 <eos>
How it works: <eos>
 <eos>
By using hardware virtualization, VBS creates a secure environment that is separate from the normal operating system. <eos>
 <eos>
This isolation makes it much harder for malware running in the main operating system to access and steal sensitive information. <eos>
 <eos>
The "LsaIso.exe" process is a key part of this VBS key isolation. <eos>
 <eos>
Key points: <eos>
 <eos>
 <eos>
 <eos>
Credential Guard relies on VBS to provide its security. <eos>
 <eos>
These features are designed to protect against advanced persistent threats and other sophisticated attacks. <eos>
 <eos>
They are primarily found in Windows Enterprise and Education editions. <eos>
 <eos>
These features need specific hardware requirements to function. <eos>
 <eos>
In essence, Credential Guard and VBS Key Isolation work together to create a more secure Windows environment by isolating and protecting sensitive credentials from potential threats. <eos>

 <eos>

 <eos>

Jak to funguje: <eos>
 <eos>
Credential Guard používá zabezpečení založené na virtualizaci (Virtualization-based Security, VBS) k vytvoření izolovaného prostředí, kde jsou tyto citlivé přihlašovací údaje uloženy a chráněny. <eos>

 <eos>
 <eos>

Účel: <eos>

VBS je hardwarově podporovaná virtualizační technologie, která vytváří zabezpečenou, izolovanou oblast paměti. <eos>
 <eos>
Izolace klíčů je součástí VBS, která konkrétně izoluje kryptografické klíče a další citlivá data. <eos>
 <eos>
Jak to funguje: <eos>
 <eos>
Pomocí hardwarové virtualizace VBS vytváří zabezpečené prostředí, které je oddělené od běžného operačního systému. <eos>
 <eos>
Tato izolace výrazně ztěžuje malwaru spuštěnému v hlavním operačním systému přístup k citlivým informacím a jejich krádež. <eos>
 <eos>
Proces "LsaIso.exe" je klíčovou součástí této izolace klíčů VBS. <eos>
 <eos>
Klíčové body: <eos>
 <eos>
 <eos>
 <eos>
Credential Guard se spoléhá na VBS, které zajišťuje jeho zabezpečení. <eos>
 <eos>
Tyto funkce jsou navrženy tak, aby chránily před pokročilými trvalými hrozbami a dalšími sofistikovanými útoky. <eos>
 <eos>
Nacházejí se především v edicích Windows Enterprise a Education. <eos>
 <eos>
Tyto funkce vyžadují ke svému fungování specifické hardwarové požadavky. <eos>

Ahoj! Ano, app.py (tvoje Flask aplikace) a dynamic_llm.py (tvůj trénovací skript) mohou běžet současně. Ve skutečnosti, aby tvůj systém pro dynamické učení fungoval tak, jak zamýšlíš, měly by běžet současně. <eos>
 <eos>
 <eos>
 <eos>
Jak by to mělo fungovat: <eos>
 <eos>
 <eos>
 <eos>
Spustíš dynamic_llm.py: Tento skript poběží na pozadí a bude neustále sledovat soubor new_data.txt. Pokud v něm najde nová data, provede dotrénování tvého modelu a uloží aktualizované váhy do model.pth. Bude také občas reagovat na tvé interaktivní zadání v konzoli pro generování textu pomocí aktuálně natrénovaného modelu. <eos>
 <eos>
 <eos>
 <eos>
Spustíš app.py (Flask): Tato aplikace spustí webový server, který bude dostupný na adrese http://127.0.0.1:5000/ (nebo jiné, pokud jsi nastavil(a) jiný port). Uživatelé mohou přes tento webový prohlížeč interagovat s tvým modelem: <eos>
 <eos>
 <eos>
 <eos>
Generovat text: Zadají počáteční sekvenci a aplikace jim zobrazí vygenerovaný text. K tomuto generování používá model (váhy) uložený v souboru model.pth a slovní zásobu z vocabulary.json. <eos>
 <eos>
Poskytovat data pro dotrénování: Zadají text do textového pole pro dotrénování a odešlou formulář. Tato data se uloží do souboru new_data.txt, který sleduje dynamic_llm.py. <eos>
 <eos>
Proč je souběžné spuštění důležité: <eos>
 <eos>
 <eos>
 <eos>
Dynamické učení: dynamic_llm.py potřebuje běžet, aby mohl reagovat na nová data v new_data.txt a provádět aktualizace modelu. <eos>
 <eos>
Webová interakce: app.py musí běžet, aby uživatelé mohli interagovat s modelem přes webové rozhraní (generovat text a poskytovat nová trénovací data). <eos>
 <eos>
Jak je spustit současně: <eos>
 <eos>
 <eos>
 <eos>

 <eos>
 <eos>
 <eos>

 <eos>
 <eos>
Nyní by měly oba skripty běžet nezávisle na sobě. Můžeš interagovat s webovou aplikací ve svém prohlížeči a zároveň sledovat výstup trénovacího skriptu v jeho terminálovém okně. Když poskytneš nová data přes webovou aplikaci, dynamic_llm.py by je měl po chvíli načíst a spustit dotrénování. Po dokončení dotrénování by se měla projevit změna v generovaném textu ve webové aplikaci (po opětovném načtení stránky nebo odeslání nového požadavku na generování). <eos>
